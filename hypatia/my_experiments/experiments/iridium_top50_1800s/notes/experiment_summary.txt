Iridium 600s 仿真实验总结

一、实验内容

本次实验模拟的是 Iridium 星座下的地面站到地面站业务承载过程，仿真周期为 600 秒，时间步长为 100 ms。

实验对象包括：
1. 66 颗 Iridium 卫星
2. 前 50 个地面站（对应文件：ground_stations_top_50.basic.txt）
3. 卫星链路模式：isls_plus_grid
4. 路由算法：algorithm_free_one_only_over_isls
5. TCP 协议：TcpBic

流量矩阵的生成方式是低秩 OD 矩阵：
1. 先用 rank=5、density=0.3、seed=123456789 生成一个 50×50 的低秩流量矩阵
2. 再将矩阵中的 OD 对映射到实际节点 ID
3. 每条流的起始时间都设为 0 ns，也就是所有流同时开始
4. 当前这次实验一共生成了 735 条流
5. 总流量约为 16.66 TB

注意：本次实验的流量规模偏大，主要原因是流数量较多、每条流大小较大，而且所有流同时在 0 时刻启动，因此 ns-3 仿真耗时较长。

二、ns-3 仿真输出文件说明

本次仿真输出位于 logs_ns3 文件夹中，主要文件如下：

1. finished.txt
   - 含义：仿真是否正常结束的标志文件
   - 数据：内容为 Yes，表示仿真已完成

2. timing_results.txt
   - 含义：仿真各阶段的耗时日志，便于人工阅读
   - 数据：包含 Configure run and logs directory、Configure simulator、Setup TCP parameters、Run simulation、Destroy simulator 等阶段，以及每个阶段的开始/结束时间和持续时间
   - 其中 Run simulation 阶段耗时约 11943.4 秒，约 3.3 小时

3. timing_results.csv
   - 含义：与 timing_results.txt 对应的结构化版本，便于程序读取和统计
   - 数据：每一行记录一个阶段名称及其耗时，单位为 ns

4. tcp_flows.txt
   - 含义：TCP 流的文本摘要，通常用于快速查看流信息
   - 数据：每条流对应一个摘要记录，包含流编号、端点、大小、状态等信息

5. tcp_flows.csv
   - 含义：TCP 流的结构化结果文件，是最重要的流级结果文件之一
   - 数据：每一行对应一条 TCP 流，包含流编号、源节点 ID、目的节点 ID、流大小、开始/结束时间、已发送或已接收字节数、结束状态等
   - 当前实验中可看到部分流状态为 NO_ONGOING 或 NO_ERR_CLOSE，表示流的结束情况不同

6. tcp_flow_0_rtt.csv
   - progress: flow_id, time_ns, bytes
   - 含义：第 0 条流的 RTT（往返时延）随时间变化记录
   - 数据：每一行包含时间戳和 RTT 值，单位通常为 ns
   - 用途：分析时延变化、链路切换、拥塞演化对 RTT 的影响

7. tcp_flow_0_cwnd.csv
   - 含义：第 0 条流的拥塞窗口（cwnd）变化记录
   - 数据：每一行包含时间戳和 cwnd 值
   - 用途：分析 TCP 拥塞控制过程

8. tcp_flow_0_progress.csv
   - rtt: flow_id, time_ns, rtt_ns
   - 含义：第 0 条流的传输进度记录
   - 数据：每一行包含时间戳和累计进度值
   - 用途：观察流随时间推进的完成情况

9. isl_utilization.csv
   - 含义：星间链路（ISL）利用率统计文件
   - 数据：每一行记录某条 ISL 在某个时间区间内的平均利用率，通常包括链路编号、时间区间和利用率数值、传输字节数
   - 用途：分析哪些星间链路更繁忙，是否存在瓶颈

三、这次实验结果的整体解读

1. 仿真已成功完成，finished.txt 显示 Yes
2. 仿真总运行时间很长，约 3.3 小时
3. 主要原因不是单一流过大，而是：
   - 流数量多（735 条）
   - 总流量大（约 16.66 TB）
   - 所有流都在 0 时刻同时启动
4. 这类设置更适合做大规模负载测试，但如果只是验证流程，建议将流量单位和密度调小
